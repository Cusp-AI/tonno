# v0.2 Plan — API redesign

## What changes

### Public API

**Before:**
```python
@autotune(configs=[KC(32, 64), KC(64, 128)], key=["N", "D"])
def fn(cfg: KC, x: Array, *, N: int = 0, D: int = 0) -> Array:
    return kernel(x, cfg.BN, cfg.BK)
```

**After:**
```python
@autotune(configs=[{"BN": 32, "BK": 64}, {"BN": 64, "BK": 128}])
@jax.jit(static_argnames=["BN", "BK"])
def fn(x: Array, BN: int = 32, BK: int = 64) -> Array:
    return kernel(x, BN, BK)
```

Gone: `cfg`, `key=`, `encode=`, `decode=`, `Concatenate` typing.  
Kept: `num_warmup`, `num_timing`.

---

## autotune's new contract

- Accepts a `jax.stages.JitWrapped` (i.e. the result of `@jax.jit`)
- `configs` is `list[dict[str, Any]]` — keys are the tunable kwarg names
- Validates at decoration time:
  - input is a jitted function
  - all keys in `configs` appear in the jit's `static_argnames`
  - all tunable kwargs have defaults in the underlying function signature
  - all configs have identical keys

---

## Cache key

Derived the same way jit does it — no separate concept:

```python
key = (
    tuple(jax.typeof(a) if isinstance(a, jax.Array) else a for a in args),
    tuple((k, jax.typeof(v) if isinstance(v, jax.Array) else v)
          for k, v in sorted(non_tunable_kwargs.items()))
)
```

`jax.typeof(x)` returns `ShapedArray(shape, dtype)` — public, stable, hashable.  
Non-tunable static kwargs (e.g. `transpose: bool`) are included by value — same as jit.

---

## Wrapper behaviour

1. Receive call `fn(x, w, transpose=True)` (no tunable kwargs)
2. Compute cache key from non-tunable args
3. Cache hit → inject best config, call jitted fn
4. Cache miss → sweep all configs, time each, store best, call jitted fn
5. If user passes tunable kwargs explicitly → bypass autotune, call jitted fn directly

---

## What stays the same

- `cache.py` — unchanged (MISSING sentinel, path traversal guard, threading lock, etc.)
- `hardware.py` — unchanged
- `num_warmup`, `num_timing` — unchanged
- JSON cache format — unchanged (fn_name, device_name, cache_key → best config)
- `fn_name` derived from `__qualname__` of the underlying function — unchanged

---

## Breaking changes

- `cfg` as first positional arg → removed
- `key=` parameter → removed
- `encode=` / `decode=` parameters → removed (configs are plain dicts, always JSON-serialisable)
- Configs must be `list[dict[str, Any]]` — NamedTuple configs no longer accepted directly

---

## Migration

```python
# v0.1
class KC(NamedTuple):
    BN: int = 32
    BK: int = 64

@autotune(configs=[KC(32, 64), KC(64, 128)], key=["N"])
def fn(cfg: KC, x: Array, *, N: int = 0) -> Array:
    return kernel(x, cfg.BN, cfg.BK)

# v0.2
@autotune(configs=[{"BN": 32, "BK": 64}, {"BN": 64, "BK": 128}])
@jax.jit(static_argnames=["BN", "BK"])
def fn(x: Array, BN: int = 32, BK: int = 64) -> Array:
    return kernel(x, BN, BK)
```
